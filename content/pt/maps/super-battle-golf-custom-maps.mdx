<Callout type="info" title="Guia Rápido">
- **Exportação no Unity**: Use o modelo de projeto específico do Unity para criar asset bundles para os **mapas personalizados do Super Battle Golf**.
- **Nomenclatura de Bundles**: Sempre use o formato `Usuario.NomeDoPacote` para evitar conflitos de arquivos na pasta de mods.
- **Configuração JSON**: Cada pacote de mapas requer um arquivo JSON correspondente para definir nomes dos buracos, pontuações de par e dificuldade.
- **Estrutura de Arquivos**: Coloque os bundles exportados no diretório `Weapon X/plugins/Super Battle maps/maps/`.

<Section icon={Settings} title="Primeiros Passos com Mapeamento Personalizado">

A criação de **mapas personalizados do Super Battle Golf** permite que os jogadores vão além do conteúdo base do jogo e projetem desafios únicos. O processo envolve o uso da engine Unity para construir níveis e, em seguida, exportá-los como asset bundles que o framework de modding do jogo possa reconhecer. Antes de começar, certifique-se de ter baixado a versão correta do modelo de projeto do Unity, pois ele contém os scripts necessários para o pipeline de build.

| Requisito | Descrição | Recomendação |
| :--- | :--- | :--- |
| **Versão do Unity** | Versão específica correspondente à build do jogo | Verifique o Discord oficial para a versão atual |
| **Modelo de Projeto** | Projeto Unity exportado para o Super Battle Golf | Essencial para o script de build |
| **Asset Bundles** | Formato usado para empacotar cenas 3D | Deve ser criado através do menu personalizado |
| **Conhecimento de JSON** | Edição básica de texto para metadados do mapa | Necessário para definições dos buracos |

<Callout type="tip" title="Organização do Projeto">

<Section icon={Map} title="Exportando Seus Mapas do Unity">

O processo de exportação é a parte mais técnica da criação de **mapas personalizados do Super Battle Golf**. Você deve distinguir entre bundles de cena e bundles de assets. Um bundle de cena contém o layout real do nível (o buraco), enquanto um bundle de assets contém modelos 3D ou texturas personalizadas que você importou para o projeto.

**Destaques do Vídeo:**
- **Nomenclatura de Bundles**: Como usar a convenção de nomenclatura `usuario.nome-do-pacote`.
- **Console de Build**: Usando o comando `B:` para verificar o conteúdo do bundle.
- **Pipeline de Exportação**: Navegando no menu de build para gerar os arquivos finais.
- **Pacotes de Várias Cenas**: Adicionando vários buracos em um único asset bundle.

<YouTubeEmbed videoId="7mkmXowrSlY" title="Super Battle Golf Custom Maps - Exporting Your Maps" />

<Callout type="warning" title="Restrição de Asset Bundle">
Você não pode misturar cenas e outros recursos (como texturas ou modelos) no mesmo bundle. Se você usar objetos personalizados que não estão no jogo base, deve criar um bundle separado (ex: `usuario.assets`) e referenciá-lo adequadamente.
</Callout>

<Steps>
  <Step num="1" title="Limpar Asset Bundles">
    Antes de compilar, vá ao menu Build no Unity e selecione "Clear All Asset Bundle Names". Isso garante que nenhum dado antigo interfira na sua nova exportação.
  </Step>
  <Step num="2" title="Atribuir Nomes aos Bundles">
    Selecione seu arquivo de cena na janela Project. No Inspector (canto inferior direito), crie um novo nome de Asset Bundle. Use o formato `SeuNome.NomeDoMapa`.
  </Step>
  <Step num="3" title="Criar (Build) os Bundles">
    Navegue até o menu "Build" e selecione "Build Asset Bundles". Este processo pode levar de 1 a 2 minutos por cena, dependendo do seu hardware.
  </Step>
  <Step num="4" title="Localizar Arquivos Exportados">
    Abra a pasta do seu projeto no Windows Explorer e navegue até `Assets/Bundles`. Você só precisa do arquivo sem extensão (ignore os arquivos `.manifest`).
  </Step>
</Steps>

</Section>

<Section icon={Target} title="Configurando os Metadados JSON">

O arquivo JSON funciona como o "Cartão de Identidade" para seus **mapas personalizados do Super Battle Golf**. Sem um JSON configurado corretamente, o jogo não saberá como exibir seu mapa no menu ou quais são as regras de pontuação. Você pode copiar o `example.json` fornecido na pasta do mod e editá-lo com qualquer editor de texto como o Notepad++.

| Chave JSON | Tipo de Valor | Descrição |
| :--- | :--- | :--- |
| **sceneName** | Texto | Deve corresponder exatamente ao nome da cena no Unity |
| **holeName** | Texto | O nome exibido para os jogadores no menu |
| **par** | Inteiro | O número padrão de tacadas para o buraco |
| **difficulty** | Texto | Easy, Medium, Hard ou Expert |
| **enabled** | Booleano | Deve ser definido como `true` para aparecer no jogo |

<Callout type="success" title="Suporte a Múltiplos Buracos">
Para adicionar vários buracos a um único pacote, basta adicionar uma vírgula após o colchete de fechamento da entrada do primeiro buraco e colar um novo conjunto de colchetes com os dados do segundo buraco.
</Callout>

</Section>
