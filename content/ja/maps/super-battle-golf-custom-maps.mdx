<Section icon={Settings} title="カスタムマッピングの始め方">

**Super Battle Golfのカスタムマップ**を作成することで、プレイヤーはベースゲームのコンテンツを超えて、ユニークなチャレンジをデザインすることができます。このプロセスでは、Unityエンジンを使用してレベルを構築し、ゲームのモッディングフレームワークが認識できるアセットバンドルとしてエクスポートします。開始する前に、ビルドパイプラインに必要なスクリプトが含まれている、正しいバージョンのUnityプロジェクトテンプレートがダウンロードされていることを確認してください。

| 要件 | 説明 | 推奨事項 |
| :--- | :--- | :--- |
| **Unityのバージョン** | ゲームのビルドに一致する特定のバージョン | 公式Discordで現在のバージョンを確認 |
| **プロジェクトテンプレート** | Super Battle Golf用にエクスポートされたUnityプロジェクト | ビルドスクリプトに必須 |
| **アセットバンドル** | 3Dシーンをパッケージ化するために使用されるフォーマット | カスタムメニュー経由でビルドする必要あり |
| **JSONの知識** | マップメタデータ用の基本的なテキスト編集 | ホールの定義に必要 |

<Callout type="tip" title="プロジェクトの整理">

<Section icon={Map} title="Unityからのマップのエクスポート">

エクスポートプロセスは、**Super Battle Golfのカスタムマップ**を作成する上で最も技術的な部分です。シーンバンドルとアセットバンドルを区別する必要があります。シーンバンドルには実際のレベルレイアウト（ホール）が含まれ、アセットバンドルにはプロジェクトにインポートしたカスタム3Dモデルやテクスチャが含まれます。

**動画のハイライト:**
- **バンドルの命名**: `username.packname` の命名規則の使用方法。
- **ビルドコンソール**: `B:` コマンドを使用してバンドルの内容を確認する方法。
- **エクスポートパイプライン**: ビルドメニューを操作して最終ファイルを生成する方法。
- **マルチシーンパック**: 単一のアセットバンドルに複数のホールを追加する方法。

<Callout type="warning" title="アセットバンドルの制限">
同じバンドル内にシーンと他のアセット（テクスチャやモデルなど）を混在させることはできません。ベースゲームにないカスタムオブジェクトを使用する場合は、別個のバンドル（例：`username.assets`）を作成し、それに応じて参照する必要があります。
</Callout>

<Steps>
  <Step num="1" title="アセットバンドル名のクリア">
    ビルドする前に、Unityの「Build」メニューに移動し、「Clear All Asset Bundle Names」を選択します。これにより、古いデータが新しいエクスポートに干渉しないようになります。
  </Step>
  <Step num="2" title="バンドル名の割り当て">
    Projectウィンドウでシーンファイルを選択します。インスペクター（右下）で、新しいアセットバンドル名を作成します。`YourName.MapName` の形式を使用してください。
  </Step>
  <Step num="3" title="バンドルのビルド">
    「Build」メニューに移動し、「Build Asset Bundles」を選択します。このプロセスは、ハードウェアのスペックによってシーンあたり1〜2分かかる場合があります。
  </Step>
  <Step num="4" title="エクスポートされたファイルの場所を確認">
    Windowsエクスプローラーでプロジェクトフォルダを開き、`Assets/Bundles` に移動します。拡張子のないファイルのみが必要です（`.manifest` ファイルは無視してください）。
  </Step>
</Steps>

</Section>

<Section icon={FileCode} title="ディレクトリ設定とファイル管理">

**Super Battle Golfのカスタムマップ**をエクスポートしたら、それらを正しいゲームディレクトリに配置する必要があります。ゲームは、Weapon Xプラグイン構造内のカスタムコンテンツを探します。配置が正しくないと、ゲーム内の「Custom Holes（カスタムホール）」タブにマップが表示されません。

| フォルダ階層 | パスの構成要素 | 目的 |
| :--- | :--- | :--- |
| **ルート** | `Super Battle Golf/` | ゲームのメインインストールフォルダ |
| **プラグインパス** | `Weapon X/plugins/` | すべてのmod関連データの場所 |
| **マップストレージ** | `Super Battle maps/maps/` | カスタムレベル用の特定のディレクトリ |
| **あなたのパック** | `15_cool_maps/`（例） | 特定のマップパック用のユニークなフォルダ |

<Section icon={Target} title="JSONメタデータの設定">

JSONファイルは、**Super Battle Golfのカスタムマップ**の「IDカード」として機能します。正しく設定されたJSONがないと、ゲームはメニューにマップを表示する方法や、スコアルールが何であるかを認識できません。modフォルダにある `example.json` をコピーして、Notepad++などのテキストエディタで編集できます。

| JSONキー | 値の型 | 説明 |
| :--- | :--- | :--- |
| **sceneName** | String | Unityのシーンの正確な名前と一致する必要があります |
| **holeName** | String | メニューでプレイヤーに表示される名前 |
| **par** | Integer | ホールの標準打数（パー） |
| **difficulty** | String | Easy、Medium、Hard、またはExpert |
| **enabled** | Boolean | ゲーム内に表示するには `true` に設定する必要があります |

<Callout type="success" title="複数ホールへの対応">
1つのパックに複数のホールを追加するには、最初のホールエントリーの閉じブラケットの後にカンマを追加し、2番目のホールのデータを含む新しいブラケットのセットを貼り付けるだけです。
</Callout>

</Section>

<Section icon={Sparkles} title="最適化と品質保証（QA）">

機能するマップを作成することは最初のステップにすぎません。コミュニティに受け入れられるためには、十分にプレイできることを確認することが不可欠です。**Super Battle Golfのカスタムマップ**は、共有する前にローカルでテストする必要があります。衝突判定（コリジョン）ゾーンやアウトオブバウンズ（OB）のトリガーには細心の注意を払ってください。これらはカスタム作成において最も一般的な失敗要因です。

<Callout type="warning" title="機能しない要素">
モッディングフレームワークの一部の機能はまだ開発中の可能性があります。特定のインタラクティブオブジェクトが機能しない場合、それはマップのデザインではなく、現在のmodバージョンの制限によるものである可能性が高いです。
</Callout>
